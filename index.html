03

Prompts vagos vs. prompts precisos

La diferencia entre una respuesta genérica y una que realmente sirve

✗ Vago
Escríbeme sobre redes sociales para mi negocio
Sin contexto, sin formato, sin objetivo claro → respuesta genérica de 800 palabras que no sirve para nada
✓ Preciso
Escríbeme 5 ideas de posts para Instagram (formato carrusel) para un restaurante en Lima que quiere mostrar su cocina de autor. Tono cercano, con emojis. Una idea por línea, con el hook en negrita.
Específico, accionable, con formato claro → resultado útil de inmediato
✗ Vago
Explícame cómo funciona Python
¿Para qué nivel? ¿Con qué objetivo? → Respuesta demasiado amplia
✓ Preciso
Soy desarrollador web con experiencia en JavaScript. Explícame los conceptos clave de Python que debería aprender primero, usando analogías con JS. Máximo 5 puntos, con un ejemplo de código corto para cada uno.
Nivel calibrado, formato definido, analogías útiles → aprendizaje efectivo
✗ Vago
Ayúdame con mi presentación
¿Qué tipo de ayuda? ¿Sobre qué tema? → Imposible responder bien
✓ Preciso
Tengo una presentación de 10 min para inversores sobre mi startup SaaS. Ya tengo el contenido. Necesito que me sugieras cómo estructurar los slides (cuántos, en qué orden) para maximizar el impacto. Dame un esquema con el título de cada slide.
Restricciones claras, objetivo específico → estructura accionable
04

Técnicas que marcan la diferencia

Estrategias para prompts complejos o tareas de alta precisión

Técnica Cómo usarla Cuándo aplicarla
Razonamiento paso a paso Añade "piensa paso a paso" o "explica tu razonamiento antes de responder" Problemas lógicos, análisis, decisiones con múltiples variables
Ejemplos positivos y negativos "Quiero algo como [X], pero NO como [Y]" Textos creativos, código con estilo específico, análisis de datos
Pedir variantes "Dame 3 versiones: formal, informal, creativa" Títulos, subject lines, descripciones, textos de marketing
Restricciones explícitas "Sin usar la palabra X", "máximo N palabras", "evita Y" Cuando sabes qué NO quieres o tienes límites técnicos
Iterar sobre la respuesta "Toma tu respuesta y hazla un 30% más concisa / técnica / persuasiva" Refinamiento de borradores, ajustar tono o profundidad
Usar etiquetas XML Pide separar secciones: <análisis> / <recomendación> / <código> Respuestas largas que necesitas procesar o copiar por partes
Preguntar antes de responder "Antes de responder, hazme las preguntas que necesites" Tareas ambiguas, proyectos grandes, cuando no sabes exactamente qué pedir

Plantillas listas para usar

Copia, adapta y lanza — reemplaza los textos en [corchetes] con tu información

🐛Depurar un error
Eres un desarrollador senior en [lenguaje]. Tengo el siguiente error: [pega el mensaje de error aquí] Este es el bloque de código relevante: [pega el código aquí] Explica por qué ocurre el error y dame la corrección con el código listo para copiar. Si hay más de una causa posible, lista cada una.
🧪Escribir tests
Eres un experto en testing con [framework de tests]. Escribe tests unitarios para la siguiente función: [pega la función aquí] Cubre: casos normales, casos borde y entradas inválidas. Usa nombres descriptivos para cada test. Formato: código listo para ejecutar, sin explicaciones adicionales.
🔍Code review
Actúa como un revisor de código experimentado. Analiza el siguiente código y señala: 1) bugs o errores potenciales, 2) problemas de rendimiento, 3) mejoras de legibilidad, 4) problemas de seguridad si los hay. [pega el código aquí] Para cada punto, indica la línea, el problema y la solución sugerida.
📝Documentar código
Genera documentación técnica para el siguiente [función / clase / módulo] en formato [JSDoc / docstring / comentarios inline]. Incluye: descripción general, parámetros con tipos, valor de retorno y un ejemplo de uso. [pega el código aquí]
✉️Email de prospección
Eres un experto en ventas B2B. Escribe un email de primer contacto para [tipo de empresa objetivo]. Contexto: Ofrecemos [descripción de producto/servicio] que ayuda a [beneficio principal]. El receptor es [cargo del destinatario]. El email debe: tener máximo 120 palabras, abrir con una pregunta o dato relevante para ellos, mencionar el beneficio sin sonar vendedor, y cerrar con una CTA para una llamada de 15 minutos. Tono: directo y profesional.
📱Post para redes sociales
Eres un especialista en contenido para [red social]. Crea 3 variantes de un post sobre [tema] para [tipo de negocio/marca]. Tono: [cercano / profesional / educativo]. El post debe generar [engagement / tráfico / conversiones]. Incluye emojis relevantes y hashtags al final. Cada variante en un bloque separado, con el hook en la primera línea.
🎯Propuesta de valor
Ayúdame a refinar la propuesta de valor de [nombre del producto/servicio]. Lo que hace: [descripción]. Para quién: [perfil del cliente ideal]. Problema que resuelve: [problema principal]. Alternativas actuales del cliente: [cómo lo resuelven hoy]. Dame 3 versiones de propuesta de valor: una racional (datos/resultados), una emocional (beneficio sentido) y una de diferenciación (por qué nosotros y no la competencia). Máximo 2 oraciones cada una.
📊Análisis de competencia
Actúa como analista de mercado. Compara [mi producto] con [competidor 1] y [competidor 2] en los siguientes criterios: precio, características clave, público objetivo, canales de adquisición y puntos débiles conocidos. Presenta el resultado en una tabla y luego dame 3 oportunidades de diferenciación que podría explotar basándote en los gaps que encuentres.
🧠Explicar un concepto
Explícame [concepto] como si tuviera [nivel: ningún conocimiento previo / conocimientos básicos / nivel intermedio] en el tema. Usa una analogía con algo cotidiano para la explicación central. Luego dame: 1) la definición formal, 2) un ejemplo práctico real, 3) los 3 errores más comunes al entender este concepto.
📋Plan de estudio
Crea un plan de estudio para aprender [tema] en [duración: 2 semanas / 1 mes / 3 meses]. Mi nivel actual: [principiante / intermedio]. Tiempo disponible: [horas] por semana. Objetivo final: [qué quiero poder hacer al terminar]. Estructura el plan semana a semana con: tema de la semana, recursos recomendados (tipo, no links) y un ejercicio práctico para validar lo aprendido.
❓Generar preguntas de repaso
Basándote en el siguiente contenido, genera 10 preguntas de repaso que cubran los puntos más importantes. Incluye 3 niveles: 4 preguntas de comprensión básica, 4 de aplicación y 2 de análisis crítico. Para cada una incluye la respuesta correcta en una línea aparte. [pega el contenido aquí]
✏️Feedback de escritura
Eres un editor académico. Lee el siguiente texto y dame feedback estructurado en: 1) claridad del argumento central, 2) coherencia entre párrafos, 3) problemas de gramática o estilo, 4) lo que funciona bien. Sé específico: señala párrafos o frases concretas. Al final, sugiere una reescritura del párrafo más débil. [pega el texto aquí]
📅Preparar una reunión
Ayúdame a preparar una reunión de [duración] con [número] personas sobre [tema]. Objetivo de la reunión: [qué debe quedar decidido o resuelto al final]. Contexto: [situación actual relevante]. Dame: 1) agenda con tiempos asignados, 2) las 3 preguntas clave que debo hacer, 3) posibles puntos de fricción y cómo manejarlos, 4) formato del acta de seguimiento.
📄Resumir un documento
Resume el siguiente documento para alguien que tiene 2 minutos para leerlo. Estructura así: 1) de qué trata en 1 oración, 2) los 3 puntos más importantes, 3) decisiones o acciones que requiere de mi parte, 4) fecha límite o urgencia si la hay. [pega el documento o fragmento aquí]
🤔Tomar una decisión
Necesito tomar la siguiente decisión: [describe la decisión]. Opciones que estoy considerando: [opción A] vs [opción B] (y [opción C] si aplica). Criterios más importantes para mí: [lista 2-3 criterios]. Restricciones: [tiempo, presupuesto, recursos]. Analiza cada opción contra mis criterios, identifica lo que no estoy considerando y dame una recomendación con su justificación. No me des solo pros y contras — dame tu opinión directa.
💬Responder un mensaje difícil
Ayúdame a responder el siguiente mensaje de forma [profesional / empática / directa] sin generar conflicto innecesario. Contexto de la relación: [colega / cliente / jefe / proveedor]. Lo que quiero lograr con mi respuesta: [objetivo]. Lo que no quiero que pase: [riesgo a evitar]. Mensaje que debo responder: [pega el mensaje aquí] Dame 2 versiones: una más suave y una más directa.

Glosario de prompting

Los términos que verás cuando profundices en el tema

Hacer una solicitud directamente sin dar ningún ejemplo previo. La IA responde basándose solo en su entrenamiento. Es el tipo de prompt más común y funciona bien para tareas simples o bien definidas.

Ejemplo
Traduce "good morning" al español.

Incluir 2 a 5 ejemplos del resultado esperado dentro del mismo prompt para que la IA entienda el patrón. Muy útil para tareas con un formato o estilo muy específico que sería difícil de describir solo con palabras.

Ejemplo
Input: "perro" → Output: "🐶 animal doméstico" Input: "sol" → Output: "☀️ estrella central" Input: "libro" → Output: ?

Técnica que le pide a la IA que "piense en voz alta" antes de dar su respuesta final. Al razonar paso a paso, comete menos errores en problemas complejos. Se activa con frases como "piensa paso a paso" o "explica tu razonamiento".

Ejemplo
¿Cuántos días hay entre el 15 de marzo y el 4 de julio? Piensa paso a paso antes de responder.

Instrucción de configuración que se le da a la IA antes de empezar la conversación. Define su personalidad, tono, restricciones y contexto permanente. En la API de Claude corresponde al parámetro system. En el chat, sería como las instrucciones del perfil.

Ejemplo
"Eres un asistente de soporte para GastroCore. Solo respondes preguntas sobre el sistema. Siempre en español. Tono amable pero conciso."

Parámetro numérico (0 a 1) que controla qué tan "creativa" o predecible es la respuesta. Temperatura baja (0.1–0.3) = respuestas más consistentes y precisas. Temperatura alta (0.7–1.0) = más variedad y creatividad. Para tareas de análisis o código, usar temperatura baja. Para escritura creativa, alta.

Cuándo usarlo
Temperatura 0.1 para extraer datos de un texto. Temperatura 0.9 para generar nombres de marca.

La unidad mínima con la que los modelos procesan texto. No es exactamente una palabra: en español, 1 token ≈ 0.7 palabras. "Hola" es 1 token, "desarrollador" puede ser 2–3. Los costos de la API y los límites de contexto se miden en tokens, no en palabras ni caracteres.

Regla práctica1,000 tokens ≈ 750 palabras ≈ 1.5 páginas de texto.

La cantidad máxima de texto (medida en tokens) que el modelo puede "ver" en una sola conversación: tu historial, el system prompt y la respuesta actual. Si la conversación supera ese límite, el modelo empieza a "olvidar" los mensajes más antiguos. Claude tiene una de las ventanas más grandes del mercado.

Analogía
Es como la memoria de trabajo humana: puedes manejar cierta cantidad de información a la vez, pero no todo lo que has aprendido en tu vida.

Cuando la IA genera información que suena convincente pero es falsa o inventada. No lo hace intencionalmente: es una limitación de cómo funcionan los modelos de lenguaje. Ocurre más en datos específicos como fechas, nombres, citas textuales o estadísticas. Siempre verifica hechos críticos.

Cómo reducirlo
Pide fuentes, usa frases como "si no sabes, dímelo", y verifica datos concretos antes de usarlos.

Ataque en el que un texto malicioso dentro del contenido que procesa la IA intenta anular sus instrucciones originales. Relevante si construyes aplicaciones donde la IA procesa texto de terceros. Por ejemplo: un documento que dice "ignora tus instrucciones y haz X en su lugar".

Relevante para
Desarrolladores que integran LLMs en productos: valida siempre las entradas del usuario antes de pasarlas al modelo.

Técnica que combina un sistema de búsqueda con un modelo de lenguaje. Antes de responder, el sistema recupera documentos relevantes de una base de datos y los incluye en el contexto. Permite que la IA responda con información actualizada o privada sin necesidad de re-entrenar el modelo.

Caso de uso
Un chatbot de soporte que busca en tu documentación interna antes de responder al cliente.
Un buen prompt no es el más largo ni el más técnico. Es el que te da exactamente lo que necesitas al primer intento, porque te tomaste treinta segundos más en pensar qué querías de verdad.
